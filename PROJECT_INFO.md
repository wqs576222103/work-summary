## 🚀 如何使用

### 方式 1：立即开始（推荐新手）

1. **获取 API Key**
   - 访问 https://console.anthropic.com/
   - 注册并创建 API Key

2. **配置 API Key**
   - 打开 `.env` 文件
   - 替换 `ANTHROPIC_API_KEY=your_api_key_here` 为你的实际 Key

3. **打开浏览器**
   - 访问：http://localhost:3000
   - 服务器已经在运行！

4. **开始生成**
   - 按照界面上的提示操作即可

### 方式 2：自定义开发

如果你需要修改或扩展功能：

```bash
# 修改代码后重启
Ctrl+C  # 停止服务
npm start  # 重新启动
```

## 📊 技术架构

### 后端技术栈
- **运行时**: Node.js
- **Web 框架**: Express.js
- **Git 库**: simple-git
- **AI SDK**: @anthropic-ai/sdk
- **环境配置**: dotenv

### 前端技术栈
- **HTML5** - 页面结构
- **CSS3** - 样式和动画
- **原生 JavaScript** - 交互逻辑
- **Fetch API** - HTTP 请求

### API 端点
| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/scan` | POST | 扫描目录获取项目列表 |
| `/api/username` | POST | 获取仓库 Git 用户名 |
| `/api/generate` | POST | 生成工作总结 |

## 🔐 安全提示

1. **保护 API Key**
   - `.env` 文件已在 `.gitignore` 中排除
   - 不要将 API Key 分享给他人
   - 定期更换 API Key

2. **权限控制**
   - 工具只读取 Git 仓库信息
   - 不会修改任何代码文件
   - 只在授权范围内使用

3. **数据安全**
   - 提交记录临时保存在内存中
   - 生成完成后自动删除临时文件
   - 不存储任何用户数据

## 🐛 已知问题

1. **API 错误**: 当前显示 "403 Forbidden" 是因为 `.env` 文件中还没有配置有效的 API Key
   - 解决方案：按上述步骤配置 API Key 即可

2. **Windows 路径**: 某些情况下 Windows 路径可能需要使用正斜杠 `/`
   - 解决方案：尝试使用 `D:/projects` 而不是 `D:\projects`

## 📞 技术支持

### 常见问题解决

**Q: 服务器启动失败？**
```bash
# 检查 Node.js 版本
node --version

# 重新安装依赖
rm -rf node_modules
npm install
```

**Q: 端口被占用？**
修改 `server.js` 中的 PORT 值：
```javascript
const PORT = 3001; // 改为其他端口
```

**Q: Git 命令执行失败？**
确保 Git 已安装并可在全局使用：
```bash
git --version
```
