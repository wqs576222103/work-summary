# 使用流程说明

## 📋 完整使用流程

### 第一步：配置环境（一次性）
```bash
# 1. 安装依赖（已完成）
npm install

# 2. 配置 API Key（在 .env 文件中）
ANTHROPIC_API_KEY=your_api_key_here

# 3. 启动服务器（已运行）
npm start
```

### 第二步：打开 Web 界面
浏览器访问：**http://localhost:3000**

### 第三步：扫描项目
1. 在"根目录路径"输入框中填写包含多个项目的父目录
   - Windows 示例：`D:\work-projects` 或 `D:/work-projects`
   - Mac/Linux 示例：`/home/user/projects`

2. 点击 **"扫描项目"** 按钮

3. 等待扫描完成，系统会列出所有找到的 Git 项目

### 第四步：选择项目
- 勾选需要生成总结的项目
- 或使用 **"全选"** / **"全不选"** 快速操作
- 可以看到已选择的项目数量

### 第五步：设置参数
1. **日期范围**：选择开始日期和结束日期
   - 默认显示最近 7 天
   
2. **分支名称**（可选）：
   - 例如：`alpha`, `main`, `master`, `develop`
   - 留空则使用当前分支
   
3. **Git 用户名**（可选）：
   - 留空则自动使用仓库配置的用户名
   - 也可以手动指定特定用户名

### 第六步：生成总结
1. 点击 **"生成总结"** 按钮

2. 等待处理完成（时间取决于项目数量和提交记录）

3. 查看生成的结果

### 第七步：使用结果
- **查看**：在结果区域查看 AI 生成的工作总结
- **复制**：点击"复制结果"按钮复制到剪贴板
- **下载**：点击"下载结果"保存为文本文件

## 💡 使用技巧

### 技巧 1：定期生成
建议每周五下午生成当周的工作总结，形成工作记录习惯。

### 技巧 2：精确筛选
- 如果多人协作一个项目，指定用户名可以只关注自己的提交
- 如果项目有多个并行开发分支，指定分支可以更精准

### 技巧 3：灵活调整
- 第一次生成不满意？调整日期范围重新生成
- 某些项目不需要？取消勾选排除掉

### 技巧 4：批量处理
- 可以同时选择十几个甚至几十个项目一起生成
- 适合负责多个项目的技术人员

### 技巧 5：结果优化
复制结果后，可以根据实际情况微调：
- 补充 AI 可能遗漏的重要工作
- 调整表述更符合公司要求
- 添加下一步工作计划

## 🎯 典型应用场景

### 场景 1：周报生成
- 日期范围：本周一到本周五
- 项目：所有参与的项目
- 用途：直接作为周报素材

### 场景 2：月度总结
- 日期范围：月初到月末
- 项目：所有项目
- 用途：月度绩效考核

### 场景 3：项目回顾
- 日期范围：项目关键阶段
- 项目：特定项目
- 用途：项目复盘或汇报

### 场景 4：离职交接
- 日期范围：在职期间
- 项目：所有参与过的项目
- 用途：工作交接文档

## ⚠️ 注意事项

1. **路径格式**：Windows 路径中的反斜杠可能需要转义
2. **网络要求**：确保能访问 Claude API
3. **权限要求**：确保有项目目录的读取权限
4. **Git 要求**：确保 Git 已正确安装和配置

---

## 🔍 故障排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 扫描不到项目 | 路径错误 | 检查路径是否正确，确认目录下有.git 文件夹 |
| 生成失败 | API Key 无效 | 检查.env 文件中的 API Key 是否正确 |
| 无提交记录 | 日期范围内无提交 | 扩大日期范围或检查用户名是否正确 |
| Git 操作错误 | Git 未安装 | 安装 Git 并配置环境变量 |
